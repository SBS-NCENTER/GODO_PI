## 1. 프로젝트 한 줄 요약

> SHOTOKU XR 크레인의 베이스가 움직였을 때, 복잡한 재캘리브레이션 없이 **자동으로 (dx, dy) 보정량을 계산**하여 Unreal Engine 합성 영상의 위치를 그대로 유지시키는 시스템을 RPLIDAR + Raspberry Pi 5 + 인하우스 AMCL 알고리즘으로 구축.

### 무엇을 해결하는가

| 항목 | 현재 (도입 전) | 도입 후 (목표) |
|---|---|---|
| 크레인 베이스 이동 시 | 운영자가 복잡한 재캘리브레이션 수행 (수십 분) | 버튼 1개 누르면 자동 (`/api/calibrate` 또는 GPIO) |
| 측정 범위 | 수동 영점 조정 | 1-shot 모드 ≤ 1-2 cm (목표), Live 모드 (~10 Hz 연속) |
| 정보 흐름 | 크레인 FreeD → Arduino → Unreal | 크레인 FreeD → **RPi5 (FreeD + LiDAR 통합)** → Unreal |
| 폴백 | 단일 경로 | 기존 Arduino R4 firmware **그대로 보존**, 라인 교체로 즉시 폴백 |

## 7. 부록

### 7.1 용어 정리

| 용어 | 의미 |
|---|---|
| **AMCL** | Adaptive Monte Carlo Localization. 현재 GODO 장치의 핵심 알고리즘으로, LiDAR 데이터를 이용해 Self Localization할 수 있도록 함. particle filter 기반 위치 추정. 5,000 particles → 200 iterations 수렴. |
| **FreeD** | SHOTOKU 크레인의 카메라 위치/방향 출력 프로토콜 (X/Y/Z/Pan/Tilt/Roll/Zoom/Focus). |
| **Hot path / Cold path** | Hot = 60 Hz 하드 데드라인 (FreeD UDP). Cold = 최대 1초 지연 OK (LiDAR + AMCL). 두 경로 사이는 lock-free seqlock으로 교차. |
| **SCHED_FIFO** | Linux 실시간 스케줄링 정책. 일반 프로세스(SCHED_OTHER)보다 절대 우선. |
| **mlockall** | 프로세스 메모리를 RAM에 고정 (page fault 제거 → jitter 감소). |
| **Seqlock** | Single-writer + N-reader 락프리 데이터 구조. Generation counter 기반. |
| **PGM / YAML** | 맵 포맷. PGM = grayscale 이미지 (벽=검정, free=흰색, unknown=회색), YAML = metadata (resolution, origin, threshold). slam_toolbox 표준. |
| **Polkit** | Linux의 fine-grained 권한 시스템. systemctl poweroff 같은 sensitive 명령을 root 아닌 사용자에게 허용할 때 사용. |
| **invariant** | 코드/문서가 영구히 지켜야 하는 규칙. 위반 시 review block. |
| **SSOT** | Single Source Of Truth. 같은 데이터를 여러 곳에 복제하지 않고 한 곳에서만 정의. |
