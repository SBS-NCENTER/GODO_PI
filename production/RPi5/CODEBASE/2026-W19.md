## 2026-05-07 01:00 KST — issue#19 P4-2-19-1..7 — EDT 2D parallelisation across the issue#11 ParallelEvalPool

### Why

Phase-0 5-min capture post-issue#11 PR #99 deploy (2026-05-06, 2989 scans on news-pi01) showed LF rebuild p50 stuck at ~40 ms while evaluate_scan dropped from 94.85 ms to 45.11 ms. LF rebuild became the dominant slice (40/87 ≈ 46% of TOTAL p50 post-fix vs 29% pre-fix). issue#19 partitions the Felzenszwalb 2D EDT across the same pool (cores 0/1/2) so both 1-D passes (column then row) parallelise by 3-way; combined issue#11 + issue#19 projection: TOTAL p50 87 → ~47 ms ≈ 21.4 Hz at p50, clearing the 18 Hz post-deploy acceptance bar with margin.

**Operator decision (2026-05-07 KST)**: ship at 13.77 Hz. The 1.43× lift matched the dev-box ctest concurrent measurement (1.4–1.5×) — production isolcpus=3 reproduced the same ceiling, suggesting EDT is **memory-bandwidth-bound** (column-major reads → cache miss heavy → 3-worker parallel saturates DDR4 bus on Pi 5 → sub-linear scaling). issue#11's evaluate_scan was compute-bound and scaled to 2.10× for the same pool; the workloads are structurally different. Forcing more lift requires NEON SIMD vectorisation, prefetch directives, or transpose-tile cache locality — distinct algorithmic work, not a re-tune of the existing parallelisation. **issue#19.5** reserved for that path.

## 2026-05-06 14:34 KST — issue#11 P4-2-11-1..6 — ParallelEvalPool ships, AMCL × pool integration, diag publisher + UDS endpoint, regression bench

### Why

Phase-0 5-min main capture (PR #96 `53453f5`) showed Live mode stuck at 7.34 Hz / 136.15 ms p50 — below the 10 Hz LiDAR rate, ~30% scan drop. Plan §2 picked Option C (Pi 5 fork-join over the per-particle eval loop, joined per AMCL step, workers pinned to CPU {0, 1, 2}) over Option B (multi-tier σ pipeline) on cost (~280 LOC vs ~700) + cache topology (single 16 MB LF read-only across workers vs 3× DRAM replication). Option C clears 10 Hz at +35% margin (~73 ms p50) while keeping cascade-jitter structurally impossible (no inter-tick worker state) and bit-equal output to the sequential path (plan §3.6 weighted_mean proof).
